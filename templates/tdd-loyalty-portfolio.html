The Journey

6 Versions, 4 Refactors

V1
Green

Simple Multiplication

Just enough code to make the first tests pass. 1 point per dollar.

def calculate_points_earned(amount: float) -> int: """Calculate points from purchase.""" return int(amount) # 1 point per dollar
V2
Refactor

Extract Constants

Remove magic numbers. Add multiplier parameter for future expansion.

# Named constant instead of magic number BASE_POINTS_PER_DOLLAR = 1.0 def calculate_points_earned(amount, multiplier=BASE_POINTS_PER_DOLLAR): return int(amount * multiplier)
V3
Green

Add Tier Support (Ugly)

If/else chain to handle tiers. Works but doesn't scale well.

# Ugly if/else - needs refactoring! if tier == "bronze": multiplier = 1.0 elif tier == "silver": multiplier = 1.5 elif tier == "gold": multiplier = 2.0 elif tier == "platinum": multiplier = 3.0
V4
Refactor

Enum + Configuration Dict

Type-safe tiers with centralized configuration. Single source of truth.

class MembershipTier(Enum): BRONZE = "bronze" SILVER = "silver" GOLD = "gold" PLATINUM = "platinum" TIER_CONFIG = { MembershipTier.GOLD: {"multiplier": 2.0, ...} }
V5
Refactor

LoyaltyAccount Class

Object-oriented design. Account tracks balance, handles earning and redemption.

@dataclass class LoyaltyAccount: customer_id: str points_balance: int = 0 tier: MembershipTier = MembershipTier.BRONZE def earn_points(self, amount): ... def redeem_points(self, points, purchase): ...
V6
Refactor

Frozen Dataclass + Configurable Rules

Immutable tier config prevents accidents. Business rules are injectable for testing.

@dataclass(frozen=True) # Immutable! class TierConfig: name: str multiplier: float redemption_rate: float perks: List[str] @dataclass class LoyaltyRules: # Injectable! min_points_to_redeem: int = 500 max_redemption_percent: float = 0.50
