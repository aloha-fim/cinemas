The Journey

From Strings to State Pattern

V1
Start

String-Based Status

Simple but error-prone. Magic strings everywhere.

self.status = "available" # Magic string! self.status = "held" self.status = "avalable" # Typo compiles!
V2
Refactor

Enum-Based Status

Type safety prevents typos. IDE autocomplete works.

class SeatStatus(Enum): AVAILABLE = auto() HELD = auto() RESERVED = auto() self._status = SeatStatus.AVAILABLE # Type-safe!
V4
Refactor

Transition Table

Declarative rules. See all valid transitions at a glance.

VALID_TRANSITIONS = { SeatStatus.AVAILABLE: {SeatStatus.HELD}, SeatStatus.HELD: {SeatStatus.RESERVED, SeatStatus.AVAILABLE}, SeatStatus.RESERVED: {SeatStatus.AVAILABLE}, }
V6
Refactor

Injectable Time Provider

Test expiration without waiting. Mock time for instant tests.

@dataclass class SeatConfig: hold_timeout_minutes: int = 5 time_provider: Callable = datetime.now # Injectable! # In tests: config = SeatConfig(time_provider=lambda: fixed_time)
V7
Refactor

State Pattern

Each state is a class. Polymorphic behavior. Easy to extend.

class SeatState(ABC): def can_hold(self) -> bool: ... def can_reserve(self) -> bool: ... class HeldState(SeatState): def can_hold(self): return False # Already held def can_reserve(self): return True
