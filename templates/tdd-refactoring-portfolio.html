The Journey

6-Step Refactoring Evolution

Red Phase

Step 1: Write Tests First

Define expected behavior before any implementation exists.

def test_matinee_price_at_2pm(self): """2:00 PM on Monday = Matinee = $8""" showtime = datetime(2024, 1, 8, 14, 0) price = get_ticket_price(showtime) assert price == 8.0
Green Phase

Step 2: Quick & Dirty Implementation

Make tests pass with the simplest possible code. Ugly is OK!

# Version 1: Works but has magic numbers def get_ticket_price(showtime): hour = showtime.hour if hour < 17: # Magic number! return 8.0 # Magic number! elif hour < 22: return 12.0 else: return 10.0
Refactor #1

Step 3: Extract Constants

Replace magic numbers with named constants. Tests stay green!

# Version 2: Constants extracted MATINEE_PRICE = 8.0 EVENING_PRICE = 12.0 EVENING_START_HOUR = 17 def get_ticket_price(showtime): if hour < EVENING_START_HOUR: return MATINEE_PRICE
Refactor #2

Step 4: Separate Concerns

Extract "what slot" from "what price". Each function does ONE thing.

# Version 4: Single Responsibility def get_time_slot(showtime) -> TimeSlot: """Determine WHAT time slot""" ... def get_ticket_price(showtime) -> float: """Get HOW MUCH to charge""" slot = get_time_slot(showtime) return PRICES[slot]
Add Feature

Step 5: Add Holiday Support

New tests for holidays, then implement. Tests guide the design.

def test_christmas_day(self): """Christmas Day = Holiday = $15""" showtime = datetime(2024, 12, 25, 14, 0) price = get_ticket_price(showtime) assert price == 15.0
Refactor #3

Step 6: Configuration-Driven Design

Replace if/else chains with a rule engine. Adding new pricing tiers requires ZERO logic changes!

# Version 6: Professional-grade, extensible class ShowtimePricing: rules: List[PricingRule] def add_rule(self, rule): """Add new pricing rule - no code changes!""" def get_price(self, showtime): """First matching rule wins"""

def get_ticket_price(showtime):
    hour = showtime.hour
    day = showtime.weekday()
    
    if (showtime.month, showtime.day) in [
        (1, 1), (12, 25), (7, 4)
    ]:
        return 15.0
    if day >= 5:
        return 14.0
    if hour < 17:
        return 8.0
    elif hour < 22:
        return 12.0
    else:
        return 10.0
